# Вводная для продолжения работы в Yandex DataLens

Финальный проект курса «Инструменты анализа и визуализации данных» (МФТИ, ИАВД-11). Дедлайн: 05.04.2026.

Выбран **Yandex DataLens** как BI-инструмент. Подготовка данных выполнена в Python (pandas). Исходный датасет — Retail.xlsx (транзакции онлайн-ритейлера подарков и товаров для дома, UK-рынок, ~950K строк за 2009-2011).

## Что уже сделано

### Подготовка данных (Python pipeline)
- Исключены грязные строки (`rnd=True`, ~37K) — содержали 100% нулевых цен, пустых описаний и нераспознаваемых строк
- Нормализация текста (trim, регистр, пробелы, маппинг стран)
- Классификация каждой строки по типу операции (sale, return, shipping, discount, bad_debt и др.)
- Дедупликация (~30K полных дубликатов)
- Построение звёздной схемы: 3 факт-таблицы + 4 измерения
- Экспорт в Excel-workbook `retail_datalens_export.xlsx` (7 листов)

### Связи таблиц (звёздная схема)

```
dim_product ──── stock_code_norm ────┐
                                     │
dim_customer ─── customer_id ────────┤
                                     ├──── FACT TABLES (x3)
dim_country ──── country_name ───────┤
                                     │
dim_date ─────── date = invoice_ ────┘
                        date
```

| Факт (FK) | Измерение (PK) | Примечание |
|---|---|---|
| `stock_code_norm` | `dim_product.stock_code_norm` | Имена совпадают |
| `customer_id` | `dim_customer.customer_id` | Имена совпадают |
| `country_name` | `dim_country.country_name` | Имена совпадают |
| `invoice_date` | `dim_date.date` | Имена отличаются, тип date у обоих |

## План дашборда (2 страницы)

### Страница 1: Операционная сводка

**KPI-карточки:**
- Выручка: `SUM([line_amount])`
- Количество заказов: `COUNTD([invoice_id])`
- Средний чек: `AVG([invoice_total])`
- Активных клиентов: `COUNTD([customer_id])`
- Доля возвратов, %

**Чарты:**
1. Линейный: выручка по месяцам (`year_month` × `SUM(line_amount)`)
2. Столбчатая: топ-10 стран без UK (фильтр `is_uk = False`)
3. Treemap / кольцевая: выручка по товарным категориям (`product_category`)
4. Столбчатая: выручка по каналам (`channel`), с разбивкой UK vs International
5. Таблица: топ-10 товаров по выручке

**Селекторы:** период, страна/регион, канал, товарная категория

**Сворачиваемый блок описания** (ОБЯЗАТЕЛЬНО по ТЗ, наверху первой вкладки) — ответы на 4 вопроса из задания.

### Страница 2: Клиентская аналитика

**KPI-карточки:**
- Идентифицированных клиентов
- Доля анонимных по выручке
- Средний LTV (Top-tier)

**Чарты:**
6. Столбчатая с накоплением: клиенты по tier (количество + выручка)
7. Scatter: клиенты — заказы × средний чек (цвет = tier)
8. Линейный: динамика новых клиентов по месяцам
9. Горизонтальная столбчатая: топ-10 клиентов по выручке
10. Столбчатая: возвраты по категориям товаров
11. Столбчатая/heatmap: выручка по регионам × кварталам

## Требования из ТЗ (ключевое)

1. Минимум **2 страницы** с навигацией
2. **Интерактивность и кросс-фильтрация**
3. **Сворачиваемый блок описания** на первой вкладке с ответами на:

   - Какие дополнительные срезы были бы полезны?

## Важные особенности данных

- **UK = 92% транзакций** — на графиках по странам лучше фильтровать UK отдельно
- **22.6% продаж анонимны** (customer_id = "ANONYMOUS") — учитывать в клиентской аналитике
- **Октябрь 2011 неполный** (данные до 27-го) — помечен флагом `is_last_incomplete_month`
- **Сезонность**: пик в ноябре-декабре (рождественские подарки)
- Метрики `invoice_total` и `invoice_item_count` предрассчитаны на каждой строке — не нужны LOD-выражения
